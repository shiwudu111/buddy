# 客户端协作规则

**版本**：V1.0  
**更新**：2026-04-13  
**面向**：所有参与 buddy-client 开发的协作者

---

## 核心原则

buddy-client 的协作必须遵守"**WSL 开发 → Windows 审核 → 老板 push**"模式。

```
WSL 本地开发
    ↓ 提交到 WSL 本地 Git
Windows 审核区
    ↓ 人工审查变更
老板 push 到 GitHub
```

任何 Agent 不得绕过人工审核直接 push。

---

## Git 协作规则

### 提交规范（WSL）
- 每次 commit 必须附清晰描述
- 格式参考：`[模块] 简短说明`
- 示例：`[Login] 拆分 AuthCoordinator 职责`
- **禁止**大而含糊的 commit，如"更新"、"fix"

### 分支策略
- `develop`：日常开发分支
- `main`：稳定分支，**只接受人工 merge**
- 所有功能开发在 develop 上进行

### 提交前自检
- [ ] `git diff --check` 无空白符错误
- [ ] `git status` 无意外文件
- [ ] 变更与任务目标一致

## Cocos Creator 规范

### Scene / Prefab 维护
- ❌ **禁止手改** `.scene` / `.prefab` 中的脚本 `__type__`
- ❌ **禁止手改** 对象 `__id__` 编号
- ❌ **禁止手改** 组件引用编号
- scene 文件必须由 Cocos Creator 编辑器维护，手改会损坏文件

### 脚本挂载
- 脚本丢失时：优先在编辑器里重新挂载
- 不要通过文本编辑修复 scene 文件引用

### 节点命名
- 节点名称约定需收敛为统一常量层（进行中）
- 参考：`Login模块说明-V1.0.md`

## API 联调规则

### 接口数据来源
- **客户端联调数据表**：`固定联调测试数据表-V1.0.md`
- **接口基线**：`03-API接口/API-MVP-BASELINE-V1.0.md`
- 固定测试账号/密码/ID 必须从本文档读取，**不得硬编码**

### 联调流程
1. 客户端准备好待测功能
2. 后端（Cipher）确认 API 可用
3. 双方对照验收清单逐项测试
4. 有争议查 `docs/contracts/` 与后端联调契约
5. 通过后更新联调验收清单

### 联调依赖
- 客户端联调依赖后端 API 就绪
- 如遇 API 问题，转交 Cipher 处理，不要自己猜测接口行为

## 协作角色边界

| 情况 | 处理方式 |
|------|----------|
| 后端 API 相关问题 | → 交给 Cipher |
| 产品需求/范围问题 | → 汇报老板 |
| Git push / 发布 | → 老板负责 |
| 客户端代码/场景/UI | → TA-OpenClaw（或当前开发者）|
| 发现文档与代码不一致 | → 先确认冻结决策，再同步更新 |
