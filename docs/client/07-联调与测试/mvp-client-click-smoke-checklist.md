# MVP 客户端点击冒烟验收清单

## 目标

验证 Cocos 客户端真实点击路径是否能贯通 MVP 主链路。API 冒烟脚本只能证明后端和服务层链路可用；本清单用于验证真实 UI 节点、按钮、状态提示、页面切换和数据刷新。

## 前置条件

- WSL 后端服务已启动，`http://localhost:3000/` 返回健康响应。
- Windows 客户端代码已同步到 WSL 或当前 Cocos 工程已加载最新 `buddy-client`。
- 运行过：
  - root：`node tools/smoke-mvp-flow.mjs`
  - client：`bunx tsc --noEmit --ignoreDeprecations 6.0`
- 准备一组新的学生账号和家长账号，避免同日作业重复提交影响奖励。

## 当前自动化状态

- 当前 Codex 环境未找到 `CocosCreator.exe`、`CocosCreator`、`Creator` 或 `cocos` 命令。
- `buddy-client` 当前没有可直接打开的 Web build 产物；`docs/client/*.html` 是静态设计壳，不等同于 Cocos 运行时。
- 因此本清单当前定位为人工点击验收清单。后续若要自动化，需要先补齐 Cocos Creator 命令行路径或生成 Web build。

## 自动化前置补齐

如需让 agent 后续自动执行 UI 点击验收，需要提供其中一种入口：

1. Cocos Creator 可执行路径，例如：
   - `C:\...\CocosCreator.exe`
2. 可运行的 Web build 输出目录，例如：
   - `buddy-client/build/web-desktop/`
3. 可启动的本地预览命令，例如：
   - `npm run preview`
   - `cocos preview`

具备上述入口后，agent 才能使用浏览器或自动化工具执行真实点击、截图和断言。

## 验收路径

### 1. 登录 / 注册

- 打开 Login 场景。
- 注册学生账号。
- 预期：
  - 注册成功提示清晰。
  - 自动进入 Main，或可通过登录进入 Main。
  - 失败时显示后端错误，不出现乱码。

### 2. 创建或进入宠物主页

- 首次进入 Main。
- 如无宠物，创建宠物。
- 预期：
  - Main 页面显示宠物名称、状态、等级、心情 / 饥饿等核心状态。
  - 无宠物时有明确创建入口。
  - 创建后 dashboard 能刷新并展示后端返回状态。

### 3. Dashboard 刷新

- 在 Main 首屏等待自动同步或触发刷新。
- 预期：
  - 不出现长时间 loading。
  - 若同步成功，状态来自 dashboard。
  - 若同步失败，提示“保留已有状态”之类说明，不伪造成功。

### 4. 提交作业获得奖励

- 打开作业中心。
- 选择数学作业。
- 上传或使用当前占位图片路径。
- 提交。
- 预期：
  - 提交成功。
  - 展示奖励信息，例如逻辑饼干。
  - 成功后触发 dashboard 或背包刷新。

### 5. 背包显示奖励

- 从作业结果页进入背包，或切换 Main 顶部/底部背包入口。
- 预期：
  - 背包中出现 `logic_cookie / 逻辑饼干`。
  - 数量与后端一致。
  - 不出现空背包但作业显示已奖励的矛盾状态。

### 6. 使用背包物品

- 点击逻辑饼干。
- 预期：
  - 后端 `/inventory/use` 成功。
  - 宠物状态刷新。
  - 背包数量减少或消失。
  - Main 反馈区显示“已使用”类真实结果。

### 7. 日记显示事件

- 打开日记页。
- 预期：
  - 先可显示 dashboard 最近事件。
  - 打开日记页后刷新 `/events` 完整列表。
  - 能看到作业、奖励、背包使用相关事件。
  - 标题和详情之间分隔清晰，不出现乱码。

### 8. 聊天发送消息

- 打开聊天入口。
- 发送一条消息。
- 预期：
  - 消息发送成功。
  - 聊天历史刷新。
  - 若后端返回失败，客户端 fallback 文案不会污染正式日记或库存。

### 9. 家长侧绑定 / 查看

- 回到 Login，注册或登录家长账号。
- 绑定刚才的学生账号。
- 查看孩子宠物状态和周报。
- 预期：
  - 绑定成功。
  - 家长侧可查看孩子宠物摘要。
  - 周报展示作业统计。
  - 家长侧只读，不直接修改孩子宠物状态。

## 通过标准

- 以上 9 个步骤都能通过真实 UI 点击完成。
- 每一步失败时都有明确错误提示或可定位日志。
- 客户端不伪造库存、不直接修改正式 pet 状态、不写入伪日记事件。
- API 冒烟和 UI 点击验收结论一致。
