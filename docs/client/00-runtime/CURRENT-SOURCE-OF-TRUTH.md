## 4. 主链路优先级

所有工作优先保证这条主链路不被破坏：

```text
登录 -> 创建宠物 -> 提交作业 -> 宠物状态变化 -> 家长查看结果
```

如果新改动影响主链路稳定，应优先回退或停止，而不是继续叠加功能。

## 5. 当前执行真相

### 5.1 Login

当前登录入口执行口径：

```text
restore -> brandEntry -> roleSelect -> authForm(role, mode)
```

固定规则：

- `roleSelect -> authForm` 默认进入 `mode=login`。
- 登录成功后的真实身份以后端返回 `user.role` 为准。
- `restore` 无会话或恢复失败时统一回到 `brandEntry`。
- 不拆多个 Cocos Scene。

### 5.2 Parent Binding

当前正式联调口径：

- `1 个家长 <-> 1 个孩子`
- 双家长绑定同一个孩子不是当前已支持能力。
- 状态码语义：
  - `404`：孩子不存在
  - `409`：孩子已被其他家长绑定，或当前家长已绑定其他孩子
  - `403`：当前登录用户不是家长

### 5.3 First Pet

当前 MVP 对首次宠物创建的正式业务结果只有：

```text
无宠物学生首次进入 -> 创建宠物 -> 进入主链路
```

当前不承诺：

- 蛋类型
- 稀有度
- 物种
- hatchSeed
- 孵化结果可复现

任何已有“选蛋 / 孵化”演出，一律视为原型或关闭能力，不进入正式验收。

### 5.4 Chat

学生端聊天 v1 已完成并收口。

当前正式口径：

- 聊天入口在 `Main` 学生端主界面内。
- 聊天只在当前运行会话内保留。
- 退出登录后清空。
- 关闭浏览器后重新启动不自动回灌旧聊天。
- 聊天不会影响宠物成长。
- `chat/history` 接口当前不作为启动自动恢复主流程。

### 5.5 Pet Growth

`Pet 成长/进化 v1` 当前正式口径：

- 只做学生端。
- 只做成长页展示与局部联动。
- 只接两个成功触点：
  - `喂养成功`
  - `作业提交成功`
- 最近成长反馈只描述最近一次结果。
- 不扫描历史作业。
- 不回放历史事件。
- 不伪造成长增量数值。
- 当前成长页功能可验收，但界面观感不作为本阶段最终视觉定版标准。

### 5.6 Main Interface

- 学生端主界面改版以 `index.html` 作为当前视觉参考。
- 主页需要真实口粮资源系统支撑，不接受前端伪库存。
- `energy / health` 作为下一阶段重要状态字段，需要与后端冻结口径对齐。
- 主界面联调应优先使用 `Main 首页 P0 结构整改执行单（当前版）`。
- 主页当前执行口径是 `app-shell + frame + top/main/bottom + left/center/right`，不再把整页缩放或 `SafeFrame` 当成首页舞台基准。
- 主页第一版已接入状态卡、宠物场景、最近事件、库存明细与选粮喂养弹层，后续只在同一条主页链路上收口优化。
- 宠物场景容器已改成真正的 `scene` 父节点，宠物主体、气泡、名字牌和底板都挂入 scene。
- 当前学生端首页已经收口到参考页的同类骨架，不再是旧版整页缩放壳子；后续只允许做同链路内的细节收口。
- 当前主页视觉只能视为“第一版功能壳子 + 初步暖色场景”，不能视为“接近参考稿完成态”。
- 当前视觉差异已单独沉淀为《Main 首页 P0 结构整改执行单（当前版）》，后续按 P0 / P1 / P2 执行，不再靠零散微调单个元素推进。

- `MainController.renderGlobalActions` 内旧的学生分支已移除，只保留家长 header action shell。
- Main 概览场景气泡、名字牌和右侧摘要卡片已使用共享产品组件集，不再使用裸 box。
- Main 概览场景背景层和宠物主体比例再次收紧，更接近参考构图。

## 2026-04-21 真相更新

- Main 现在使用分层响应式布局策略，不再依赖单一固定画板。
- 渲染层拆分为 backdrop、shell、safe area 和 interaction entry。
- shell aspect 根据 viewport breakpoint 选择并插值，使平板和手机可以复用同一套屏幕逻辑。
- 内部参考入口按钮定位在 safe area 内，便于未来内容扩展，不再硬编码屏幕尺寸。
- 后续 Main 协作必须继续使用 `viewport + breakpoint + safe area`，不得重新把固定 4:3 或固定 16:9 画板作为唯一布局模型。
