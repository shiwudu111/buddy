# 学生端 AI 对话 / 宠物对话 v1
## 本轮任务
只实现：
1. Main 内聊天 UI
2. 本地预设回复
3. 会话历史前端保留
4. 空输入 / 防重复提交
5. 后端 chat/history 联调
本轮进度：Today 下 5 项已完成，登录退出清空与全量测试补齐也已完成；通过 TypeScript 编译验证，聊天相关手工点测已通过。当前收尾口径为：聊天功能验收通过；宠物成长页功能已落地，但当前界面观感仅视为功能版，不作为 MVP 最终视觉定版，后续如需提升需单独开 UI 优化任务。
## 摘要
这轮只做 **学生端** 的宠物对话，不新增独立页签，入口放在 **Main 主界面左下角输入框 + 气泡回复** 这条路径里。  
实现方式按你确认的口径走：**两者都要**，也就是 **本地预设回复 + 后端接口联调** 一起做；UI 上展示完整会话历史，回复策略用 **简单规则 + 预设话术**，不碰复杂 AI 生成。

## 关键变化
- 在 `Main` 的学生端主界面里加一个对话区，保留现有 `宠物总览 / 作业中心 / 宠物成长`，不新增新 tab。
- 新增对话相关的前端模块，建议独立成 `ui/chat`，负责：
  - 输入框、发送按钮、消息气泡、滚动容器
  - 会话状态、发送中状态、失败回退
  - 历史消息渲染
- 新增对话服务层，负责：
  - 调后端 chat 接口保存和拉取会话
  - 生成本地预设回复作为兜底
  - 简单规则选话术：关键词 + 宠物心情 + 最近若干轮上下文
- 数据口径：
  - **UI 展示本次会话全部历史**
  - **规则引擎只看最近 3 轮上下文**
  - **只做学生端，不给家长端开放对话入口**
- 交互边界：
  - 空输入不发送
  - 发送中禁用重复提交
  - 后端成功时用后端回复入历史
  - 后端失败时用本地预设回复兜底，避免聊天中断
  - 切换主界面功能时不破坏已有聊天记录，退出登录才清空本次会话

## Test Plan
- **基础冒烟**
  - 输入一句普通话术，能看到“用户气泡 -> 宠物气泡”连续出现。
  - 输入为空时不发送。
  - 连续快速点击发送不会重复发两次。
- **规则回复**
  - 命中关键词时返回对应预设回复。
  - 宠物心情不同，回复语气能有轻微变化。
  - 连续多轮对话时，回复能参考最近几轮上下文。
- **后端联调**
  - chat 接口可正常发消息并拉历史。
  - history 接口能恢复本次会话全部记录。
  - 后端失败或超时时，前端能自动切到本地预设回复，不白屏、不阻塞。
- **会话边界**
  - 切换主界面其他功能后返回，对话记录仍在。
  - 退出登录后重新登录，本次会话对话记录清空。

## 假设
- 这轮只覆盖 **学生端**，不做家长端对话页。
- 后端按 MVP 冻结文档里的 **预设对话 + 历史记录** 口径推进；如果字段名或返回形状和文档有轻微漂移，由前端 `ChatService` 做适配，不让 UI 直接依赖后端细节。
- 不新增独立场景，不改 `Login / Parent / Homework / First-pet` 的稳定链路，只在 `Main` 内部加对话能力。
