## 已跑通的流程

### 登录流程
- [x] 登录页 → 输入账号密码 → 点击登录
- [x] 登录成功 → 保存 Token → 跳转 Main
- [x] 已登录用户打开 Login → 自动跳转 Main

### Main 页面
- [x] Main 显示 "Welcome, {username}"
- [x] 无宠物时显示 "No pet yet. Create one!" + 按钮
- [x] 有宠物时显示宠物名称、等级、饱食度、心情
- [x] 点击 "Create Pet" → 调用 API → 显示宠物
- [x] 点击 "Feed Pet" → 调用 API → 刷新状态
- [x] 登出 → 清除 Token/pet_id → 返回 Login

## 当前风险点

| 风险 | 级别 | 说明 |
|------|------|------|
| UI 节点位置 | 低 | 编程式创建的节点位置是固定值，可能需要在 Cocos Editor 中拖拽调整 |
| 喂养逻辑 | 低 | updatePetResources 的 delta 值（+15饱食度/+5心情）是预设的，可能不符合后端实际逻辑 |
| API 服务依赖 | 中 | 需要确认 localhost:3000 上 buddy-server 已启动 |
