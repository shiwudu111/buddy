# 开发进度报告 - 2026-04-02 家长端修复说明

## 背景
本轮围绕家长端 MVP 主链路进行可用性修复与界面整理，目标是解决以下问题：
- 家长端“今日作业状态”与“本周周报”区域不清晰，测试时难以判断是否已展示
- 周报和作业信息过长时会超出卡片区域，无法完整查看
- 绑定孩子输入框旁边出现异常 `label` 文本，影响界面可信度

本轮重点不是扩功能，而是把家长端当前已有能力整理到“可看清、可联调、可继续冒烟”的状态。

## 本轮主要变更

### 1. 家长端首页改为上下分区
- 保留左侧“家长绑定与孩子状态”区域
- 右侧明确拆成两个信息区：
  - 上区：孩子状态与今日作业
  - 下区：本周周报
- 不再把所有家长信息混在一个区域里展示

涉及文件：
- `assets/scripts/ui/main/MainController.ts`

### 2. 今日作业与周报展示结构收敛
- 家长端状态展示改为更紧凑的摘要结构
- 今日作业状态按语文 / 数学 / 英语分别展示
- 周报摘要压缩为“周范围 / 总作业数 / 平均分 / 学科统计 / 宠物周状态”
- 避免把接口返回内容直接堆成难以阅读的大段文本

涉及文件：
- `assets/scripts/utils/format.ts`
- `assets/scripts/ui/main/MainController.ts`

### 3. 长内容改为可上下滚动查看
- “孩子状态与今日作业”区域已改为可滚动文本区
- “本周周报”区域已改为可滚动文本区
- 当文本内容超出卡片高度时，可通过滚轮或拖动继续查看，不再被卡片截断

涉及文件：
- `assets/scripts/ui/common/runtime/RuntimeUI.ts`
- `assets/scripts/ui/main/MainController.ts`

### 4. 输入框异常默认文本修复
- 运行时创建 `EditBox` 时，清理默认子节点
- 重新挂接项目自己的文本与占位文本
- 修复绑定孩子输入框旁边额外出现的异常 `label`

涉及文件：
- `assets/scripts/ui/common/runtime/RuntimeUI.ts`
