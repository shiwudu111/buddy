# 学伴精灵开发环境配置指南

## 1. 系统要求

### 1.1 硬件要求
- **CPU**: 双核以上（推荐4核）
- **内存**: 8GB以上（推荐16GB）
- **磁盘空间**: 20GB可用空间
- **网络**: 稳定的互联网连接

### 1.2 软件要求
- **操作系统**:
  - Windows 10/11 (WSL2推荐)
  - macOS 10.15+
  - Ubuntu 20.04+/CentOS 8+
- **Node.js**: v20.x LTS
- **Docker**: v24.x + Docker Compose v2.x
- **Git**: v2.x
- **数据库客户端** (可选):
  - pgAdmin 4 / DBeaver
  - Redis Insight

## 2. 环境搭建步骤

### 2.1 第一步：安装基础工具

#### Windows用户
```powershell
# 1. 安装WSL2（如果尚未安装）
wsl --install

# 2. 安装Docker Desktop
# 下载地址：https://www.docker.com/products/docker-desktop/

# 3. 安装Node.js
# 下载地址：https://nodejs.org/en/download/

# 4. 安装Git
# 下载地址：https://git-scm.com/download/win
```

#### macOS用户
```bash
# 1. 安装Homebrew（如果尚未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. 安装工具
brew install node@20
brew install docker docker-compose
brew install git

# 3. 启动Docker Desktop
open /Applications/Docker.app
```

#### Linux用户 (Ubuntu)
```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 3. 安装Docker
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker
sudo usermod -aG docker $USER

# 4. 安装Git
sudo apt install -y git

# 5. 重新登录使docker组生效
newgrp docker
```

### 2.2 第二步：安装pnpm和开发工具

```bash
# 全局安装pnpm
npm install -g pnpm

# 验证安装

# ============ JWT配置 ============
JWT_SECRET=your_jwt_secret_key_here_change_in_production  # 必须修改！
JWT_EXPIRES_IN=7d
JWT_REFRESH_SECRET=your_refresh_secret_key_here_change_in_production  # 必须修改！
JWT_REFRESH_EXPIRES_IN=30d

# ============ 文件上传配置 ============
MAX_FILE_SIZE=10485760  # 10MB
UPLOAD_PATH=./uploads
ALLOWED_FILE_TYPES=image/jpeg,image/png,image/gif

# ============ CORS配置 ============
CORS_ORIGIN=http://localhost:3000,http://localhost:5173
CORS_CREDENTIALS=true

# ============ 日志配置 ============
LOG_LEVEL=debug
LOG_FORMAT=combined
LOG_DIR=./logs

# ============ 速率限制 ============
RATE_LIMIT_WINDOW_MS=900000  # 15分钟
RATE_LIMIT_MAX_REQUESTS=100
```

## 5. 环境验证

### 5.1 验证安装
创建验证脚本 `scripts/verify-env.js`:
```javascript
const { execSync } = require('child_process');

console.log('🔍 验证开发环境...\n');

const checks = [
  { name: 'Node.js', cmd: 'node --version', expected: 'v20' },
  { name: 'pnpm', cmd: 'pnpm --version', expected: '9' },
  { name: 'Docker', cmd: 'docker --version', expected: 'Docker version' },
  { name: 'Docker Compose', cmd: 'docker-compose --version', expected: 'Docker Compose version' },
  { name: 'Git', cmd: 'git --version', expected: 'git version' }
];

checks.forEach(check => {
  try {
    const output = execSync(check.cmd, { encoding: 'utf8' }).trim();
    const passed = output.includes(check.expected);
    console.log(`${passed ? '✅' : '❌'} ${check.name}: ${output}`);
  } catch (error) {
    console.log(`❌ ${check.name}: 未安装`);
  }
});

console.log('\n📋 环境验证完成！');
```

运行验证：
```bash
node scripts/verify-env.js
```

## 6. 故障排除

### 6.1 常见问题

#### 问题1: Docker启动失败
```bash
# 检查Docker服务状态
sudo systemctl status docker

# 重启Docker服务
sudo systemctl restart docker

# 清理Docker资源
docker system prune -a
```

#### 问题2: 端口冲突
```bash
# 查看端口占用
sudo lsof -i :3000
sudo lsof -i :5432
sudo lsof -i :6379

# 停止占用进程或修改端口
# 修改 .env 和 docker-compose.yml 中的端口配置
```

#### 问题3: 数据库连接失败
```bash
# 检查数据库服务
docker-compose ps

# 查看数据库日志
docker-compose logs postgres

# 重启数据库服务
docker-compose restart postgres

# 重置数据库
pnpm prisma:migrate reset
```

#### 问题4: Redis连接失败
```bash
# 检查Redis服务
docker-compose ps

# 测试Redis连接
docker-compose exec redis redis-cli ping

# 重启Redis服务
docker-compose restart redis
```

### 6.2 开发工具问题

#### ESLint/Prettier不工作
```bash
# 重新安装依赖
rm -rf node_modules
pnpm install

# 重新生成配置文件
pnpm eslint --init
```

#### TypeScript类型错误
```bash
# 清理构建缓存
rm -rf dist

# 重新构建
pnpm build

# 检查TypeScript配置
npx tsc --noEmit
```

## 7. 生产环境准备

### 7.1 生产环境配置
创建 `.env.production`:
```env
NODE_ENV=production
PORT=3000

# 生产数据库（使用云服务）
DATABASE_URL=postgresql://prod_user:strong_password@prod-db.example.com:5432/study_companion_prod

# 生产Redis
REDIS_URL=redis://:strong_password@prod-redis.example.com:6379

# 生产JWT密钥（必须使用强密钥）
JWT_SECRET=very_strong_production_secret_key
JWT_REFRESH_SECRET=very_strong_production_refresh_secret_key

# 生产文件存储（使用对象存储）
UPLOAD_PROVIDER=oss  # oss/cos/s3
OSS_ACCESS_KEY_ID=your_access_key
OSS_ACCESS_KEY_SECRET=your_secret_key
OSS_BUCKET=study-companion
OSS_REGION=oss-cn-hangzhou

# 生产日志
LOG_LEVEL=info
LOG_FORMAT=json
```

### 7.2 Docker生产构建
```bash
# 构建生产镜像
docker build -t study-companion:prod .

# 运行生产容器
docker run -d \
  --name study-companion-prod \
  -p 3000:3000 \
  --env-file .env.production \
  study-companion:prod
```

## 8. 团队协作规范

### 8.1 Git工作流
```bash
# 1. 创建功能分支
git checkout -b feat/add-pet-system

# 2. 开发并提交
git add .
git commit -m "feat: 添加宠物系统基础功能"

# 3. 推送到远程
git push origin feat/add-pet-system

# 4. 创建Pull Request
# 在GitHub/GitLab上创建PR，请求代码审查

# 5. 合并到主分支
# 通过PR合并，保持提交历史整洁
```

### 8.2 代码规范
- 使用TypeScript，禁止使用any类型
- 遵循ESLint和Prettier配置
- 编写单元测试，覆盖率>80%
- 提交前运行 `pnpm lint` 和 `pnpm test`
- 使用有意义的提交信息

### 8.3 文档要求
- 每个API接口必须有Swagger文档
- 复杂业务逻辑必须有代码注释
- 数据库变更必须有迁移脚本
- 新功能必须有使用文档

## 9. 快速开始脚本

创建 `scripts/setup.sh`:
```bash
#!/bin/bash

echo "🚀 学伴精灵开发环境快速设置脚本"

# 检查必要工具
command -v node >/dev/null 2>&1 || { echo "❌ 请先安装Node.js"; exit 1; }
command -v pnpm >/dev/null 2>&1 || { echo "❌ 请先安装pnpm"; exit 1; }
command -v docker >/dev/null 2>&1 || { echo "❌ 请先安装Docker"; exit 1; }

# 安装依赖
echo "📦 安装依赖..."
pnpm install

# 生成Prisma客户端
echo "🔧 生成Prisma客户端..."
pnpm prisma:generate

# 启动Docker服务
echo "🐳 启动Docker服务..."
pnpm docker:up

# 等待服务启动
echo "⏳ 等待服务启动..."
sleep 10

# 运行数据库迁移
echo "🗄️ 运行数据库迁移..."
pnpm prisma:migrate dev --name init

# 启动开发服务器
echo "🚀 启动开发服务器..."
pnpm dev &

echo ""
echo "✅ 设置完成！"
echo "📊 服务状态："
echo "   - 应用: http://localhost:3000"
echo "   - API文档: http://localhost:3000/api-docs"
echo "   - 数据库管理: http://localhost:5555"
echo "   - pgAdmin: http://localhost:5050 (用户名: admin@study.companion, 密码: admin123)"
echo ""
echo "📋 常用命令："
echo "   pnpm dev          # 启动开发服务器"
echo "   pnpm test         # 运行测试"
echo "   pnpm lint         # 代码检查"
echo "   pnpm docker:logs  # 查看服务日志"
echo "   pnpm prisma:studio # 打开数据库管理"
```

给脚本执行权限并运行：
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```
