# 主界面视觉还原整改清单-V1.0

> 已停用：这份文档仅保留历史背景，不再作为当前首页执行依据。
> 当前执行依据请查看 `Main 首页 P0 结构整改执行单（当前版）.md`。

## 文档定位
- 编写时间：2026-04-20
- 适用阶段：学生端主界面改版第一轮视觉审查后
- 目标：把“方向正确但还原度不足”的问题，拆成可执行的视觉整改项
- 使用方式：本清单只服务学生端 `Main` 主界面视觉还原，不替代资源系统联调文档

## 审核结论
当前学生端主界面已经进入“暖色三栏主页”方向，但离参考稿 `index.html` 的完成度仍有明显差距。

当前版本更接近：
- 功能壳子已搭起来
- 主页结构已初步成立
- 宠物场景、最近事件、底部操作区已经有第一版落地

当前版本仍然不接近：
- 参考稿的整体外壳层级
- 顶部品牌区和胶囊导航的气质
- 中间舞台区的经验卡 + 场景 + 宠物的一体化效果
- 右侧日志区的主次层级和彩色卡片系统
- 底部操作区与聊天区的整合关系
- 宠物角色的软萌厚度和设计完成度

结论口径：
- 功能层：可继续推进
- 视觉层：暂不视为 `index.html` 级别还原完成
- 后续应按本清单逐块整改，而不是继续零散微调单个元素

## P0：必须先收住的结构问题

### 1. 顶部壳子结构不对
当前问题：
- 真实界面顶部更像“居中页面标题 + 左账号 + 右按钮”
- 缺少参考稿里的品牌块、胶囊导航和整体顶部卡片结构

整改目标：
- 顶部先变成完整的主页壳子，而不是普通页面标题栏
- 标题、账号、页签、右侧操作进入同一视觉系统

验收标准：
- 顶部区域能一眼看出“宠物主页壳子”而不是后台页标题
- 标题不再悬空成为视觉主角
- 页签、品牌、操作区有统一卡片和留白逻辑

### 2. 中间经验卡缺失
当前问题：
- 中间主场景缺少参考稿里很关键的经验卡 / 进度卡
- 场景区因此少了一个“上半区信息锚点”

整改目标：
- 在中间主舞台上方补齐经验进度卡
- 经验卡要承担“成长感”和“视觉分层”的作用

验收标准：
- 中间区形成“经验卡 -> 场景 -> 名字牌”的完整垂直结构
- 经验展示不再只是散落在左侧状态里

### 3. 底部操作区没有形成统一底栏
当前问题：
- 按钮像是独立摆放的彩色块
- 和聊天区、场景下沿互相争空间
- 没有参考稿里完整底栏 card 的包裹感

整改目标：
- 底部主操作区统一收进一个独立容器
- 聊天区和底部按钮区关系重新梳理，不再互相覆盖

验收标准：
- 底部区域一眼能看出“操作栏”
- 按钮排列有统一边距、统一节奏、统一圆角体系
- 不再出现按钮和聊天区打架、被切边、被压住的问题

### 4. 聊天区没有融进主页
当前问题：
- 当前聊天输入区仍然保留明显工具感
- 和整页暖色、奶油、玩具感的主界面语言冲突

整改目标：
- 聊天区要成为主页的一部分，而不是外挂在主页上的深色工具条
- 输入框、标题、计数、发送按钮进入同一套暖色视觉体系

验收标准：
- 聊天区与底部操作区、主场景风格一致
- 输入框不再用明显突兀的深黑系
- 宠物回复气泡、聊天输入、右侧日志三者有更自然的联动关系

### 5. 右侧日志区结构不像参考稿
当前问题：
- 现在右栏仍偏“信息列表”
- 缺少参考稿里的主日志卡、辅助信息、节奏层级

整改目标：
- 日志区从“堆文本”改成“有主次关系的互动日志模块”
- 需要有主事件卡、辅助说明、稳定的彩色标签系统

验收标准：
- 最近事件区不再只是若干同质卡片
- 至少能明显区分主事件和辅助事件
- 文本密度、字号、留白达到可读状态，不再拥挤

## P1：视觉语言层面的主要差距

### 6. 外壳层级不够
当前问题：
- 页面缺少参考稿那种“外层大壳 + 内层柔卡 + 局部子卡”的层层包裹关系

整改目标：
- 补强主页外壳、主内容容器、子卡片之间的层级差
- 通过圆角、边距、明暗层次建立“玩具盒 / 宠物屋”感

### 7. 左侧状态区像列表，不像卡片系统
当前问题：
- 状态项仍然偏“文本 + 进度条”
- 缺少图标、缺少统一内边距、缺少卡片气质

整改目标：
- 每一项状态都成为完整的小卡
- 建立统一的标题、数值、轨道、图标布局

验收标准：
- 左侧状态区不再像报表列表
- 统一的 stat-item 视觉规律清晰

### 8. 场景氛围还不成立
当前问题：
- 有太阳、云、花草，但仍像若干装饰物拼在一起
- 场景层之间没有形成统一空间关系

整改目标：
- 把天空层、装饰层、地面层、宠物舞台层做成真正一体化的舞台
- 让宠物“站在场景里”，而不是“摆在场景前”

验收标准：
- 中间区的第一视觉印象是“宠物乐园场景”
- 装饰不再零散，地面和宠物关系自然

### 9. 色彩系统不统一
当前问题：
- 暖色底成立了，但局部蓝色、黑色、粉色、橙色之间还没有统一明度规则
- 一些局部颜色过硬，跳出了整页语境

整改目标：
- 统一品牌橙、奶油白、薄荷绿、蓝紫、粉色的使用边界
- 把强对比色压缩到重点区域，不再满屏同时发声

验收标准：
- 页面整体颜色更克制
- 主色、辅助色、强调色职责清晰

### 10. 文字层级没有拉开
当前问题：
- 标题、副标题、说明、标签、正文还在同一视觉声量附近

整改目标：
- 重做标题系统和文本节奏
- 保证模块标题、提示说明、事件正文、辅助文案各有清晰层级

验收标准：
- 页面阅读顺序更稳定
- 不再出现“每块都像重点”的情况

## P2：角色与装饰细节精修项

### 11. 宠物造型还不够软萌完整
当前问题：
- 现在更像圆角图形拼出的吉祥物
- 还不到参考稿那种完整角色设计感

整改目标：
- 继续优化头身比例、脸部结构、腮红、肚皮、耳朵和爪子的关系
- 增加角色厚度、重心和抱团感

### 12. 对话气泡不够自然
当前问题：
- 气泡位置偏硬
- 仍有“覆盖层”感，不像宠物正在说话

整改目标：
- 调整气泡位置、尺寸、圆角、尾巴关系
- 让气泡成为角色的一部分，而不是叠在场景上的卡片

### 13. 名字牌、花草、阴影还缺完成度
当前问题：
- 名字牌还偏平
- 花草、云朵、阴影还停在“有元素但不够精”

整改目标：
- 补强名字牌挂件感
- 统一装饰元素的圆润度、厚度和细节级别

## 建议整改顺序
1. 先做顶部壳子和底部操作栏重构
2. 再补中间经验卡和场景容器层级
3. 接着重做右侧日志区结构
4. 然后再把聊天区融进主页
5. 最后精修宠物角色、气泡、名字牌和装饰细节

## 非目标说明
本清单当前不处理：
- 家长端视觉重构
- 聊天功能扩展
- 聊天影响成长
- 新 tab
- 复杂 AI 表现
- 后端资源系统规则改写

本清单也不等同于“现在立即全做完”。它的作用是：
- 让协作方看到当前差距在哪
- 避免把“已有第一版”误判成“接近设计稿”
- 为下一轮具体 UI 实施提供优先级顺序
