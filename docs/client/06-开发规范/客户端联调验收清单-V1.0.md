# 客户端联调验收清单 V1.0

## 文档信息
- **整理时间**：2026-04-19
- **适用阶段**：MVP 联调期
- **适用对象**：客户端、后端、测试、项目负责人

## 目标
用于统一当前 MVP 阶段客户端联调验收口径，确保学生端与家长端在关键主链路、异常态和提示文案上达到可交付的最小标准。

## 验收结果标记建议
- 通过
- 失败
- 部分通过
- 被后端阻塞
- 待确认

## 一、学生端验收

### 1. 登录与会话
- [通过] 可使用真实学生账号登录
- [通过] 登录后可进入学生端首页
- [通过] 退出登录后可返回登录页
- [通过] 重新登录后会话恢复正常

### 2. 宠物链路
- [通过] 无宠物账号可看到创建宠物入口
- [通过] 创建宠物成功后可显示宠物核心状态
- [通过] 已有宠物账号可正常显示宠物名称、等级、饥饿度、心情值、经验值
- [待确认] 失效 `petId` 可自动回收，不会卡死在错误状态
- [通过] 无本地宠物映射时，刷新提示真实，不误报成功

### 3. 作业链路
- [通过] 可进入作业中心
- [通过] 学科切换正常
- [通过] 各学科草稿独立保存
- [通过] 提交作业后有明确成功 / 失败提示
- [通过] 今日作业状态可见
- [通过] 作业历史可见，且内容完整

### 4. 学生端聊天
- [通过] `Main` 内可看到聊天 UI
- [通过] 空输入不发送
- [通过] 连续点击不会重复发送
- [通过] 本地预设回复和后端回复都可正常显示
- [通过] 切换 `宠物总览 / 作业中心 / 宠物成长` 后，本次会话聊天记录仍在
- [通过] 退出登录后重新登录，旧聊天记录不会继承
- [通过] 关闭浏览器后重新启动，不会自动回灌旧聊天记录

### 5. 宠物成长 / 进化 v1
- [通过] 学生端可进入 `宠物成长` 页签
- [通过] 可看到当前状态：阶段、等级、经验、饥饿、心情、状态
- [通过] 可看到进化条件：当前阶段、下一阶段、等级要求、进化时机、保守判断
- [通过] 喂养成功后，成长页可看到最近成长反馈
- [通过] 作业提交成功后，成长页可看到最近成长反馈
- [通过] 聊天不会影响成长
- [通过] 进化接口失败时不白屏，会显示保守提示
- [说明] 当前成长页仅按功能版验收通过，不作为 MVP 最终视觉定版

### 6. 提示与稳定性
- [通过] 刷新提示与真实结果一致
- [通过] 空态与失败态提示可读
- [通过] 页面无明显卡死、白屏、错误跳转

## 二、家长端验收

### 1. 登录与空态
- [通过] 可使用真实家长账号登录
- [通过] 登录后可进入家长端首页
- [通过] 未绑定状态展示清晰
- [通过] 未绑定时点击刷新，提示为“当前尚未绑定孩子，请先绑定后再刷新”

### 2. 绑定链路
- [通过] 支持按 `child_username` 绑定
- [通过] 支持按 `child_id` 绑定
- [通过] `child_id` 口径已确认为孩子账号 `User.id`
- [通过] 绑定成功后能加载孩子状态
- [通过] 绑定结果在重新登录后仍可恢复
- [通过] 当前已绑定其他孩子时，前端提示中文且符合 `409` 语义
- [通过] 孩子已被其他家长绑定时，前端提示中文且符合 `409` 语义
- [通过] 孩子不存在时，前端提示中文且符合 `404` 语义

### 3. 查看链路
- [通过] 可查看孩子宠物状态
- [通过] 未创建宠物的孩子可给出明确中文提示
- [通过] 可查看今日作业状态
- [通过] 可查看周报区域
- [通过] 周报接口未完成时可显示明确留声提示

### 4. 界面与交互
- [通过] 右侧信息已按上下分区展示
- [通过] 今日作业区域支持上下滚动
- [通过] 周报区域支持上下滚动
- [通过] 不再直接在前端暴露孩子 ID
- [通过] 输入框旁边无异常 `label`
- [通过] 控制台不再出现 `Can't add renderable component to this node because it already have one.`

### 5. 刷新与会话
- [通过] 已绑定孩子场景下，刷新结果真实且稳定
- [通过] 退出登录后可返回登录页
- [通过] 重新登录后可恢复家长端状态

## 三、当前正式联调口径
- [通过] 当前正式只支持：`1 个家长 <-> 1 个孩子`
- [通过] 双家长绑定同一个孩子不属于当前缺陷，而是未支持能力
- [通过] 绑定相关状态码语义已对齐：
  - `404`：孩子不存在
  - `409`：孩子已被其他家长绑定，或当前家长已绑定其他孩子
  - `403`：当前登录用户不是家长
- [通过] 学生端聊天当前不作为启动自动恢复主流程
- [通过] 学生端聊天不会影响宠物成长

## 四、通过标准建议
满足以下条件，可判定当前客户端达到本阶段联调验收标准：
1. 学生端主链路通过
2. 家长端主链路通过
3. 学生端聊天 v1 功能通过
4. 宠物成长 / 进化 v1 功能通过
5. 关键空态与异常态提示可读
6. 当前冻结口径下无阻塞性前端问题

## 五、备注
- 若字段、路径或状态码与当前基线冲突，以冻结后的接口口径为准。
- 若环境切换，请先更新固定联调测试数据表，再执行本清单。
- 宠物成长页当前观感问题单独归入后续 UI 优化任务，不阻塞当前功能验收。
- 学生端主界面改版与口粮资源系统仍处于联调准备阶段，当前不纳入“已通过”判定；对应契约请以 `Main 首页 P0 结构整改执行单（当前版）.md` 为准。
