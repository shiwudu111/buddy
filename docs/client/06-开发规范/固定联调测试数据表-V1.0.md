# 固定联调测试数据表 V1.0

## 文档信息
- **整理时间**：2026-04-03
- **适用阶段**：MVP 联调期
- **适用对象**：客户端、后端、测试

## 目标
用于固定当前联调阶段的测试账号、测试身份、关键标识和推荐用途，避免团队在回归时反复临时找号、猜数据、混用环境。

## 使用原则
- 优先使用本表中的固定账号做联调与回归。
- 若账号失效、密码变更、环境切换，必须同步更新本表。
- `child_id` 一律指孩子账号的 `User.id`，不是宠物 ID。
- 当前家长绑定正式口径为：`1 个家长 <-> 1 个孩子`。

## 二、当前阶段建议最少固定的 6 组数据
1. 1 个学生端正常登录账号
2. 1 个学生端已有宠物账号
3. 1 个学生端无宠物账号
4. 1 个家长端未绑定账号
5. 1 个家长端已绑定账号
6. 1 组绑定冲突验证账号与对应孩子账号

## 四、推荐维护方式
- 由客户端或测试维护文档结构。
- 由后端确认 `child_id` 与真实 `User.id` 是否一致。
- 每次环境切换或批量重建测试数据后，先更新此表，再开始回归。

## 五、注意事项
- 不要把宠物 ID 当成 `child_id` 使用。
- 不要把旧环境导出的 ID 直接拿到新环境复用。
- 如果按 `child_id` 绑定返回 `404`，优先检查该值是否真的是孩子账号 `User.id`。
- 若涉及真实密码或敏感环境数据，建议在团队内部版本中维护；当前仓库可保留结构与字段，不强制提交明文账号密码。
