## 请后端优先配合的接口

### P0：认证

#### 1. 注册
```http
POST /api/v1/auth/register
```

**客户端请求体**
```json
{
  "username": "student123",
  "password": "SecurePass123",
  "role": "CHILD"
}
```

**需要后端保证**
- 成功返回 `token`
- 返回 `user` 对象
- `user.role` 使用：`CHILD | PARENT`

**建议响应**
```json
{
  "success": true,
  "data": {
    "user": {
      "id": "uuid",
      "username": "student123",
      "role": "CHILD",
      "childId": null
    },
    "token": "jwt"
  }
}
```

#### 2. 登录
```http
POST /api/v1/auth/login
```

**客户端请求体**
```json
{
  "username": "student123",
  "password": "SecurePass123"
}
```

#### 3. 获取当前用户
```http
GET /api/v1/auth/me
```

**需要后端保证**
- token 有效时返回统一用户结构
- token 无效时返回 401

## P0：宠物

### 4. 创建宠物
```http
POST /api/v1/pets
```

**客户端请求体**
```json
{
  "name": "Buddy"
}
```

### 5. 获取宠物状态
```http
GET /api/v1/pets/{petId}
```

**客户端依赖字段**
```json
{
  "pet_id": "uuid",
  "name": "Buddy",
  "level": 1,
  "hunger": 100,
  "mood": 100,
  "experience": 0,
  "status": true
}
```

**注意**
- 数据库内部可以是 `fullness / growth`
- 但给客户端的字段请统一映射为 `hunger / experience`

### 6. 更新宠物资源
```http
PATCH /api/v1/pets/{petId}/resources
```

**客户端请求体**
```json
{
  "fullness_delta": 15,
  "mood_delta": 5,
  "growth_delta": 0,
  "reason": "manual_feed"
}
```

**请后端明确**
- 是否接受 `manual_feed`
- `reason` 可选还是必填

## 字段统一要求

### 1. 用户角色
请统一使用：
```text
CHILD | PARENT
```

不要混用：
- `child`
- `parent`
- `student`
- `admin`

### 2. 宠物字段
客户端展示字段统一为：
- `hunger`
- `mood`
- `experience`

数据库内部字段可保留：
- `fullness`
- `mood`
- `growth`

但请在 API 返回层做映射，不要把内部字段直接透出给客户端。

### 3. 家长绑定字段
请统一：
- 用户对象中的孩子关联字段：`childId`
- 不要再混用 `parentId` 表达“家长绑定的孩子”

## 错误码和错误响应

客户端当前最少需要如下约定：

### 推荐统一格式
```json
{
  "success": false,
  "message": "错误描述"
}
```

### 关键状态码
- `400` 参数错误
- `401` token 无效 / 未登录
- `403` 无权限
- `404` 资源不存在
- `409` 重复创建 / 重复提交
- `429` 频率限制

## 当前阻塞项

### 阻塞 1：家长绑定字段未冻结
客户端当前做了兼容发送，但这不是长期方案，需要后端尽快确认正式字段。

### 阻塞 2：用户对象中的孩子关联字段未统一
当前客户端会兼容：
- `childId`
- `parentId`

但正式方案应只保留一个口径。

### 阻塞 3：作业提交是否长期走 JSON 未确认
当前客户端先按文本联调走通主链路，后续如果升级到 `multipart/form-data`，请后端提前通知。

### 阻塞 4：宠物资源接口请求字段与返回字段存在内外命名差
客户端已按当前基线处理，但后端必须明确这是否为长期方案。
