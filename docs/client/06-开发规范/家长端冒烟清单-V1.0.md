# 家长端冒烟清单 V1.0

## 文档信息
- **整理时间**：2026-04-02
- **适用阶段**：MVP 联调期
- **适用对象**：客户端测试、联调同学、家长端功能回归

## 目标
确认家长端核心 MVP 主链路是否可用：

`登录 -> 进入家长端 -> 绑定孩子 -> 查看孩子宠物状态 -> 查看周报留壳 -> 刷新 -> 退出登录`

## 测试前准备
- 使用真实可登录的家长账号。
- 准备一个已存在的学生账号，用于绑定。
- 如需复测绑定流程，先确认该家长账号是否已绑定孩子。
- 如需验证冲突场景，准备一个已被其他家长绑定的学生账号。

## 一、进入家长端

### 1. 登录进入家长端
检查项：
- 能使用家长账号登录。
- 登录成功后进入“学伴精灵客户端 · 家长端”。
- 页面不是空白，无明显报错。

完成标准：
- 家长端主界面正常显示。

### 2. 顶部操作
检查项：
- “刷新”按钮可点击。
- “退出登录”按钮可点击。
- 当前账号信息显示正常。

完成标准：
- 顶部操作不是死按钮。

## 二、绑定孩子

### 3. 未绑定状态展示
检查项：
- 页面提示当前尚未绑定孩子。
- 可见孩子标识输入框。
- 可见“绑定孩子”按钮。

完成标准：
- 未绑定状态信息清晰。

### 4. 按孩子账号绑定
操作：
- 输入 `child_username`
- 点击“绑定孩子”

检查项：
- 是否出现明确提示：成功 / 失败 / 冲突。
- 成功后是否显示当前已绑定孩子。
- 刷新后绑定结果是否仍然存在。

完成标准：
- 绑定成功时，页面可看到已绑定结果。

### 5. 按孩子 ID 绑定
操作：
- 输入 `child_id`
- 点击“绑定孩子”

检查项：
- 是否支持按 `child_id` 绑定。
- 返回结果是否与按账号绑定一致。

完成标准：
- 两种绑定口径至少有一种稳定可用，并与后端冻结口径一致。

### 6. 绑定冲突场景
建议验证以下任一场景：
- 当前家长已绑定孩子，再次绑定其他孩子。
- 当前孩子已被其他家长绑定，再次尝试绑定。

检查项：
- 是否返回明确失败提示。
- 是否不会把界面状态错误改成“已成功绑定”。

完成标准：
- 冲突场景下前端反馈真实，不误报成功。

## 三、查看孩子状态

### 7. 孩子宠物状态
前提：
- 家长账号已成功绑定孩子。

检查项：
- 可看到宠物名称。
- 可看到等级。
- 可看到饥饿度。
- 可看到心情值。
- 若后端已返回经验值，确认展示是否合理。

完成标准：
- 家长端可读到孩子宠物核心状态。

### 8. 今日作业状态
检查项：
- 可看到 `today_homework` 对应内容。
- 已提交学科有分数或对应状态。
- 未提交学科为空或未提交。

完成标准：
- 今日作业状态显示正常，不出现明显错位或空白异常。

## 四、查看周报留壳

### 9. 周报接口留壳展示
检查项：
- 可看到周报区域。
- 若接口已返回固定结构，可看到：
  - 周范围
  - 总作业数
  - 平均分
- 若接口尚未完全实现，前端应有明确提示，而不是空白。

完成标准：
- 周报区域能稳定展示结果或稳定展示留壳提示。

## 五、刷新与会话

### 10. 刷新按钮
操作：
- 点击“刷新”

检查项：
- 页面提示是否真实反映刷新结果。
- 若孩子未绑定，不应误报全部刷新成功。
- 若接口成功，应保持页面稳定。

完成标准：
- 刷新提示真实，不误导。

### 11. 退出登录
操作：
- 点击“退出登录”

检查项：
- 是否返回登录页。
- 返回后是否清空当前会话。
- 使用原账号重新登录后，是否仍能回到家长端。

完成标准：
- 退出登录流程正常。

## 六、结果记录建议

每项测试建议标记以下结果之一：
- 成功
- 失败
- 部分成功
- 被后端接口阻塞
- UI 展示异常

## 七、本轮最少需要拿到的结论
- 家长账号是否可正常进入家长端
- 是否可成功绑定孩子
- 是否能正确处理绑定冲突
- 是否能查看孩子宠物状态
- 是否能查看今日作业状态
- 是否能看到周报留壳
- 刷新与退出登录是否正常

## 备注
- 当前阶段不追求家长端界面精细度，优先保证主链路稳定可测。
- 若接口文案或字段展示异常，优先记录接口返回与前端提示是否一致。
