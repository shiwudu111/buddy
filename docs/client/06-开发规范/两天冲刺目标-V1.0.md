# 两天冲刺目标-V1.0

## 一、文档信息
- **适用仓库**：`buddy-client`
- **版本**：V1.0
- **定位**：两天内的短期冲刺目标与验收边界
- **更新时间**：2026-04-14

## 二、背景
当前项目已进入 MVP 主链路收口阶段。

已确认稳定或基本收口的链路包括：
- `Login` 主入口流程
- `Main` 学生端主界面
- 学生端首次宠物创建链路
- 家长端绑定孩子链路

当前更适合推进的工作，不是继续扩新模块，而是把现有主链路做成“可验收、可回归、可交付”的状态。

## 三、两天冲刺总目标

本轮冲刺的目标只有一个：

**把当前最关键的主链路收口到可验收状态，并为下一阶段的宠物成长功能做好实施边界准备。**

具体落点为：
- `Homework` 链路收口
- 家长端最小冒烟验证
- `Pet 成长/进化 v1` 的实施准备
- 入口与主链路的回归稳定

## 四、冲刺范围

### 4.1 本轮重点推进
1. `Homework` 收口到验收级别
   - 提交
   - 草稿
   - 切科目
   - 提交后刷新
   - 返回总览

2. 家长端最小冒烟
   - 绑定孩子
   - 查看孩子状态
   - 查看周报
   - 刷新
   - 退出登录
   - 重新登录

3. `Pet 成长/进化 v1` 实施准备
   - 明确成长节点边界
   - 明确状态字段落地范围
   - 明确演出与正式业务的边界

4. 入口与主链路回归
   - `Login`
   - `Main`
   - `Parent`
   - `Homework`
   - `First-pet`

### 4.2 本轮不碰的内容
- `AI 对话 / 宠物对话`
- 更复杂的宠物演出和抽卡包装
- 非必要视觉扩展
- 横向性能和监控大改
- 新增入口层需求

## 五、两天内的执行目标

### 第 1 天
#### 目标
把 `Homework` 和家长端的核心链路验证清楚，避免主链路数据断点。

#### 任务
- 完成 `Homework` 收口验收
- 跑一轮家长端最小冒烟
- 记录明显的联调或回归问题

#### 第 1 天完成标准
- `Homework` 的提交、草稿、切科目、提交后刷新、返回总览能顺利跑通
- 家长端绑定、查看状态、查看周报、刷新、退出登录、重新登录顺利跑通
- 没有新增 P0 / P1 阻断问题

### 第 2 天
#### 目标
把宠物成长的下一步边界和入口主链路回归整理清楚，为后续开发铺路。

#### 任务
- 输出 `Pet 成长/进化 v1` 的实施准备边界
- 再跑一轮入口与主链路回归
- 汇总冲刺结果，形成下一阶段推进建议

#### 第 2 天完成标准
- `Pet 成长/进化 v1` 有清晰的实施边界
- `Login / Main / Parent / Homework / First-pet` 的关键冒烟点没有回归
- 冲刺结束后可以明确进入下一模块，而不是继续在当前链路上打补丁

## 六、验收口径

本轮冲刺完成的最低标准是：

1. `Homework` 主链路验证通过。
2. 家长端最小冒烟验证通过。
3. `Pet 成长/进化 v1` 的实施边界冻结清楚。
4. 入口与主链路没有新增阻断问题。
5. 不再新增入口层需求，不再扩散到非必要模块。

## 七、风险控制

1. 任何会把状态写回共享单例、并可能跨会话污染数据的改动，都必须加会话边界或延后提交。
2. `Login` 和 `Parent` 只做验证和收口，不再新增需求分支。
3. `Homework` 以“提交、草稿、切科目、提交后刷新、返回总览”为唯一收口目标，不扩视觉，不扩新交互。
4. `First-pet` 继续保持“创建宠物”正式口径，不把原型演出升格为正式业务承诺。
